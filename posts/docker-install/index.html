安装 Docker

发表于 2018-12-18 | 更新于 2019-11-04 | 分类于 Docker

本文介绍怎么在 Windows 中只安装 Docker Client，在 Ubuntu 中安装 Docker，至于 Docker 的原理和机制，在以后的章节会配合 Docker 命令再细说。

为什么选择 Windows 作为 Docker 的客户端，因为本人是 .Net 程序员，而 .Net 的开发工具 Visual Studio 要支持 Docker 需要提供 Docker 命令行。

很多教程都是在同一台电脑上直接安装 Docker 的客户端和服务端，然后就开始讲解 Docker 的各种操作，
这样不能很好地体验 Docker 是使用传统的C/S架构模式(Cilent-Server)，用户是通过 Docker Client 与 Docker Daemon 建立通信。

为什么选择 Ubuntu，因为 Ubuntu ，因为 Docker 社区版本对 Ubuntu 的支持最完善

Platform x86_64 / amd64 ARM ARM64 / AARCH64 IBM Power (ppc64le) IBM Z (s390x)
CentOS ✔ ✔
Debian ✔ ✔ ✔
Fedora ✔ ✔
Ubuntu ✔ ✔ ✔ ✔ ✔

以上表格来自 官方文档

.Net 拥有了跨平台的能力，那么作为 .Net 程序员学习 Linux 是不错的技术投资，在众多 Linux 版本中我推荐 Ubuntu 或者 CentOS，
理由是 Ubuntu 敢于加入新技术，而 CentOS 是 RHEL(Red Hat Enterprise Linux) 的克隆版本。
RHEL 是很多企业采用的 Linux 发行版本，需要向 Red Hat 付费才可以使用，并能得到付过费用的服务和技术支持和版本升级。
CentOS 可以像 RHEL 一样的构筑 Linux 系统环境，但不需要向 Red Hat 付任何的产品和服务费用。因为是企业级版本，所以保守，但很稳定。

Docker客户端与服务端的通信

我们有两种方式和Docker和服务端守护进程通信

  1. 调用Docker的remote-api来和Docker的守护进程通信
  2. 用Docker的客户端与Docker服务端守护进程通信，也就是用socket，有三种socket连接方式
    • 第一种是：直接连接本地的socket文件 unix:///var/run/docker/sock
    • 第二种是：tcp://host:prot
    • 第三种是：fd://socketfd

第一种也是默认的方式，当我们在Ubuntu输入 docker run hello-world 也就是使用第一种方式，
为了让其他电脑上的Docker客户端与Docker服务端守护进程通信，我们需要开启第二种方式。

修改 /etc/docker/daemon.json 文件中的配置，如果没有则创建。

registry-mirrors 是配置仓库地址，鉴于国内网络问题，国内从 Docker Hub 拉取镜像有时会十分缓慢，此时可以配置镜像加速器。

为了不想每次都输入-H参数，那么可以在客户端机器加上环境变量
接下来，在Win系统上需要添加 环境变量

  1. 右击 我的电脑 (win10是 此电脑)
  2. 属性
  3. 高级系统设置
  4. 环境变量
  5. 点击 新建
  6. 变量名(N) 填入：DOCKER_HOST
  7. 变量值(V) 填入：tcp://192.168.26.128:2375
  8. 确定

192.168.26.128 是我虚拟机上Ubuntu的IP地址，请自行修改。

打开一个新的 cmd，输入 docker info

查看 Docker 的客户端和服务端的版本

我们发现 Docker 的客户端比服务端的低了一个版本，貌似现在Docker官方没有再单独发布 docker.exe 了,
而是改成发布一个叫 Docker Toolbox 的软件，可以到官方下载 Docker Toolbox for Windows，
安装完毕后，修改 环境变量 变量名为 DOCKER_HOST 的值为 tcp://192.168.26.128:2375

至此，Windows 上 Docker 客户端与 Ubuntu 上的 Docker 安装完毕。

结束语

为什么一定要在 Windows 上安装 Docker 客户端？
你可以在 Visual Studio 2017 上创建一个.Net Core 项目，
然后右击该项目 添加 → Docker 支持 → 目标系统 Linux
这个时候项目下就会多一个 Dockerfile 文件，
可以看看文件内容，是不是每个单词都懂，但连起来就不知道什么意思了，这是我们以后要学习的内容。
现在右击 Dockerfile 文件，选择 生成 Docker 映像，
VS2007 就会调用 Docker 的客户端命令在 Ubuntu 系统上生成镜像了。
